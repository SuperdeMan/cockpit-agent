## 1. 背景（为什么做）

R3.1 落地会话鉴权后，gRPC 仍全 `insecure`（AGENTS.md 自认「唯一遗留生产缺口」）。审计「推进建议」：
**T3.1+T3.2 齐即「安全链路无已知缺口」**。本卡给**服务间 gRPC** 加双向 TLS（mTLS），全 env 门控、
默认关（`GRPC_TLS` 未设/off = insecure，逐字保持现状）。**范围仅服务间 gRPC**；HMI↔edge-gateway 的
WS/HTTP 明文不在本卡（部署级 HTTPS/WSS，另论）。

## 2. 关键设计决策：单张共享 mesh 证书 + name override

agents 用 `socket.gethostname()`（docker 容器 ID，**动态**）注册 endpoint，被 AgentClient 拨号——
若按 hostname 校验证书，SAN 需含动态 ID，不可行。故：
- **单张共享 mesh 证书**（CN/SAN=`cockpit-mesh` + localhost/127.0.0.1），CA 自签，**所有服务共用**为
  server + client 双身份（含 `serverAuth,clientAuth` 双 EKU）。
- 客户端把校验目标名固定为 `GRPC_TLS_SERVER_NAME`（Python `grpc.ssl_target_name_override` channel
  option / Go `tls.Config.ServerName`），**无视拨号 authority**（容器 ID / service 名 / dns:/// 都行）。
- 一举解决动态 hostname + 免枚举 ~20 个 service 名进 SAN。mTLS 仍成立（CA 互信 + 强制并校验客户端证书）。

**杠杆点**：Python 所有 client channel 走单一工厂 `runtime/grpcio.py::aio_channel`（改一处全覆盖），
server 走 `aio_server()` + 端口绑定（7 处）；Go 凭证集中到新共享包 `gateway/tlscfg`。

1. **`scripts/gen-certs.ps1` + `.sh`（新）** — openssl 生成 `certs/{ca,server}.{crt,key}`，server SAN=
   `DNS:cockpit-mesh,DNS:localhost,IP:127.0.0.1` + 双 EKU。`certs/*` gitignore、`certs/.gitkeep` 保留。
2. **`runtime/grpcio.py`** — env `GRPC_TLS`/`GRPC_TLS_CA|CERT|KEY`/`GRPC_TLS_SERVER_NAME`；`_tls_enabled()`；
   lru_cache 的 `_channel_creds()`（`ssl_channel_credentials`）/`_server_creds()`（`ssl_server_credentials(...,
   require_client_auth=True)`）；`aio_channel` 开启走 `secure_channel`（+`ssl_target_name_override`）；新
   `bind_port(server, addr)`（开启走 `add_secure_port` 否则 `add_insecure_port`）。
3. **7 处 server 绑定** `add_insecure_port` → `bind_port`（`agents/_sdk/server.py` 覆盖所有 agent +
   registry/memory/llm-gateway/payment-gateway/orchestrator{edge,cloud}）；**修 1 处 stray**
   `llm-gateway/http_server.py`（裸 `grpc.aio.insecure_channel` → `aio_channel`）。
4. **`gateway/tlscfg/tls.go`（新共享包）** — `Enabled()`/`ClientCreds()`（RootCAs+ServerName 固定）/
   `ServerCreds()`（ClientCAs + `RequireAndVerifyClientCert`）；edge/cloud `main.go` 各加 `transportCreds()`
   用于 3 dial，cloud server `grpc.NewServer` 开启时追加 `grpc.Creds(ServerCreds)`。（`COPY gateway` 已含新包。）
5. **`deploy/docker-compose.yaml`** — `*python-env` anchor + 4 个显式 env 服务（registry/edge-orchestrator/
   两 Go 网关）加 `GRPC_TLS`(默认 off)+cert 路径+`GRPC_TLS_SERVER_NAME`；`x-certs-vol` anchor 挂
   `../certs:/certs:ro` 到 **19 个 gRPC mesh 服务**（基础设施/前端/collector 不入 mesh、不挂）。
6. **`.gitignore`**（`certs/*.crt|key|pem|csr|srl`）、**`.env.example`**（mTLS 段）。

### 不做 / 取舍
- 共享证书非量产（量产应 per-service 证书 + SPIFFE/轮换）；本卡是「机制 + 全栈可加密」最小闭环。
- `certs/` 无条件挂载（GRPC_TLS 关时代码不读、无害），换取免每服务条件挂载逻辑。
- HMI WS 明文（范围外）。不改 proto、不改编排核心逻辑（仅 main.py 换端口绑定调用）。
