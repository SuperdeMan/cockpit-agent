## 1. 背景

架构 §8/§12.3 要求端云走**持久多路复用 bidi 长连**；现实 `orchestrator/edge/cloud_client.py`
**每个上云请求新建一条 bidi 流**（`Connect()`→`hello`→`request`→收 `final`→关闭）——每请求多一次
hello 握手 RTT，文件头自认「Phase 2」。

探索确认（决定方案）：
- **云侧 `gateway/cloud/main.go::channelServer.Connect` 已完整支持持久多路复用**（每 `UpFrame_Request`
  起独立 `go handleRequest` 按 corrID 隔离、`sendMu` 串行下行、`ping/pong`、幂等 `MarkIfNew(corrID,10min)`、
  `edge_call`/`edge_result` 经 `pending` 多路复用）。**edge 侧改持久化无需动云侧。**
- `gateway/edge/main.go::ChannelClient`（持久客户端）**从未实例化**＝死代码（D2）；真实链路 HMI—WS→
  edge-gateway—`EdgeOrchestrator.Handle`→ edge-orchestrator(Python)—bidi→ cloud-gateway。
  **持久通道归属 edge-orchestrator**（非架构图的 edge-gateway，A3 漂移，本次补记文档、不重构拓扑）。

## 2. 方案（持久化在 Python 侧，逐字对标 Go 参考实现）

### `orchestrator/edge/cloud_client.py`（核心重写）
`CloudClient` 保留为门面（`handle()` async-generator 契约不变 = drop-in），内部升级为**进程内单条
持久 bidi**：
- **连接生命周期**：懒启动后台 `_run()`——`stub.Connect()` 建一条流、发 `Hello`（vehicle_id 取首个
  请求的 `context.vehicle_id`，回退 env `VEHICLE_ID`/`v1`）、收 `HelloAck`、起 `_reader`+`_pinger`；
  断开按指数退避+抖动（500ms→30s，对标 Go `connectLoop`）重连。
- **多路复用**：`_pending: dict[corr_id → asyncio.Queue]`；`handle()` 注册 Queue、经 `_send_lock`
  写 `UpFrame_Request`、从 Queue yield event 到 `final`、`finally` 注销。`corr_id = f"{session_id}-{uuid4}"`。
- **`_reader`**：`event`→按 corrID 投队列（`final` 由消费者识别并收尾）；`edge_call`→注入 executor
  执行、回写 `edge_result`（同 corrID）；`pong`→清 missed；`proactive`→仅记日志（proactive→HMI 走 NATS，
  通道 Proactive 不在本次范围）；`read()` 返回 EOF/异常→触发重连。
- **`_pinger`**：15s `Ping`；连丢 >3 Pong → `stream.cancel()` 触发重连。
- **断连快速失败**：断开时给所有在途队列投 `_FAIL` 哨兵 → `handle()` 消费到即 `raise` → server.py 既有
  `except` 出降级话术。
- 复用 `runtime/grpcio.aio_channel`（keepalive）；保留 `stub_factory`/`edge_call_executor` 注入点。

### `orchestrator/edge/main.py`（生命周期）
`run_aio_server(..., on_shutdown=servicer.cloud.aclose)` 优雅关闭后台任务。

### `gateway/edge/main.go`（删死代码）
删 `ChannelClient` 及其 `NewChannelClient`/`Start`/`State`/`Request`/`connectLoop`/`connect`/`pingLoop`/
`recvLoop` + 仅其用的 `channelState`/`pendingRequest`/常量 + 因此不再用的 import。保留 `handleWS`→
`EdgeOrchestrator.Handle` 实链路与 Proactive 的 NATS/WS 广播。本地无 Go 工具链 → `docker compose build
edge-gateway` 验证编译。
