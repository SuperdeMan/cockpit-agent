# 2026-07-19 赛事域两题：进球明细查询缺位 + 预测指代错场次/完赛仍预测

> 状态：已落地（本文档随实现同批提交）。
> 触发：泓舟真机四 badcase（会话 demo-aqv0mt，2026-07-19 世界杯季军赛完赛日）——
> ① `8e23ce3012ecb80c` / `acb356765b70bcdb`「这场比赛都有谁进球」只回了当日比分汇总，
> 没查进球明细；② `f3d362092d98ebda` / `bfb5d9c7fe9d65e5`「明天世界杯有什么比赛」→
> 「这场比赛你预测谁会赢」被关联到错误的比赛（把当天已完赛的法英季军赛当成决赛对阵
> 出预测），且提问时季军赛已有结果仍在预测——上下文关联 + 时效双败。

## 1. 病灶盘点（collector 真实 trace）

| 轮 | 用户 | 系统行为 | 根因 |
|---|---|---|---|
| `8e23ce30` | 「这场比赛都有谁进球」 | 重播当日比分汇总 | `_do_sports` 两坑叠加：「都有」命中 `_LIST_HINT` 误判成列表诉求吞掉详情分支；`_pick_fixture` 只认「第N场/队名」，解析不了「这场」指代（当日仅 1 场也不兜底） |
| `acb35676` | 「这场比赛详情帮我看看，都有谁进球」 | 同上 | LLM 本已规划 `info.search`（查「法国 4-6 英格兰 进球球员」，方向正确），被 route_hint(a) 按设计 replace 回 `info.sports`——hint 没错，错在 handler 该接住却没接住（`_match_detail`/events 能力**一直存在**，只是这两坑到不了） |
| `f3d36209` | 「明天世界杯有什么比赛」 | 「07-20 决赛共1场，未开赛1场。」 | 本轮不算错但埋雷：**未开赛场次不点名对阵**，语音端听不到「谁 vs 谁」，下游追问时 planner 只能从更早历史里缝合 |
| `bfb5d9c7` | 「这场比赛你预测谁会赢」 | 检索「2026 FIFA世界杯决赛 **法国 vs 英格兰** 预测」出一屏季军赛预测 | planner 把历史里的法英（季军赛）与上一轮的「决赛」缝合成幻觉对阵；hint(d) 要求联赛/阶段词共现，「这场比赛你预测…」接不住 → LLM 自由发挥；且季军赛当天已 4-6 完赛（本会话轮1 自己播过），仍被当未来赛预测 |

## 2. 修复（全部在 `agents/info/` + manifest，未改编排核心）

### 2.1 进球明细（badcase ①，`handlers/sports.py::_do_sports`）

- 新 `_GOAL_DETAIL_HINT`（进球/谁进/射手/得分/战报/赛况/集锦/经过）**强详情词优先于
  列表词**：`picked and (强详情 or not 列表)` ——「都有谁进球」的「都有」不再劫持；
  弱详情词（怎么样/详情）不夺权，「赛程怎么样」仍是列表。
- 「这场/那场」纯指代 + **当日唯一一场** → 就是指代对象（`_ANAPHOR_HINT` 或详情词命中
  时 `picked = fixtures[0]`），进 `_match_detail` 拉 api-football events（射手+分钟，
  结构化真实数据）。多场同日仍保守回列表，不猜。

### 2.2 未开赛播报点名对阵（badcase ② 根因链前端）

赛程汇总 speech 的「未开赛N场」补对阵名（≤4 场逐一点名，≤2 场附开球时刻）：
「未开赛1场：西班牙 vs 阿根廷（03:00 开球）」。语音端可听见对阵，下游追问的
对话历史里也有了真对阵可依。

### 2.3 预测指代解析重构（badcase ②，`_predictive_anchor` → `_resolve_predictive`）

统一收口 `_predictive_redirect`（`info.sports` 与 planner 直路由 `info.search` 两入口
共用；`_search` 头部对预测句先过收口，`skip_sports=True` 防回环）：

- **焦点日期链**：本句显式日期词 > **最近赛事轮用户句的焦点日期**（新
  `_focus_date_from_history`：只扫用户轮——assistant 播报里的「今天/07-20」是转述
  口径，会把焦点拽回播报当天）> 今天 → 明天。「明天世界杯有什么比赛」→「这场你
  预测谁会赢」的「这场」由此锚到明天的决赛，不再是今天的季军赛。
- **阶段过滤**：`_stage_in`（决赛/半决赛/季军赛…长词优先防「半决赛」含「决赛」子串）
  命中时按 `_round_zh(round)` 过滤场次——「决赛你看好谁」不会锚到当天季军赛。
- **完赛短路（时效面）**：指代明确（指代词/阶段词/显式日期任一）且解析到唯一一场
  **已完赛** → 直接 `_match_detail` 报结构化赛果+进球，speech 前缀「这场比赛已经踢完了，
  结果是：」，**不做任何检索预测**（系统持有的事实不让检索/LLM 猜——墙钟直答同一原则）。
  泛问（「世界杯你看好谁夺冠」）不具体，完赛场次只跳过、绝不抢答成单场结果。
- **锚点带时效框定**：未开赛场次锚点附「（MM-DD HH:MM 开球，尚未开赛）」，合成 LLM
  拿到权威时间线，真栈答案首句即「这是一场尚未开赛的比赛（7月20日凌晨3:00开球）」。
- **锚定成功即重建 query**：原话 + 结构化对阵，**丢弃** planner 缝合的幻觉对阵；
  解析不出返回 None，保留原 query 继续（不降级 LLM 可能正确的改写）。

### 2.4 hint(d) 扩召回（`manifest.yaml`）

补「(这|那)场 × 预测词」与「预测词 × (这|那)场」两个 alternation——「这场比赛你预测
谁会赢」无联赛词也确定性进 `info.sports`（planner 掷硬币面收口）；guard 扩 官司/诉讼/
选举/辩论/网球/电竞/雨/雪/事故/发布会/演出 等非足球「这场」语境。`_search` 侧收口是
第二道网：hint 漏掉、LLM 直路由 `info.search` 时同样锚定。

### 2.5 真栈复验抓到的二层缺口（planner 污染 query 通道）

首轮真栈复验：时效变体「今天这场比赛你预测谁会赢」仍出预测——planner 这次把 query
槽改写成「2026世界杯决赛 西班牙 vs 阿根廷 预测」，`_resolve_predictive` 从 query+raw
合并文本提阶段词，「决赛」把当天季军赛**阶段过滤清空**，完赛短路失效。定论：
**planner 改写的 query 是不可信的指代通道**（badcase ② 的法英缝合与此同源）——
指代词/日期词/阶段词只从**用户原话**（`ref_text`）取；query 仅参与联赛识别
（封闭词表，无幻觉面）。该形态已钉进单测（污染 query + 原话「今天这场」→ 报赛果）。

## 3. 验证

- 单测：info 200 passed（+3：进球指代双句式 / 焦点日期锚定双入口 / 完赛短路带污染
  query）；orchestrator/cloud 223 passed；`test_sports_nearby_routing` +2（指代×预测
  召回 / guard 反例）；`eval_route_hints` 87/87（+3 例）。全量见 AGENTS.md §4。
- 真栈（重建 info-agent 容器，WS 原句回放，session demo-fix53574）5/5：
  1. 「这场比赛都有谁进球」→ 十粒进球逐一射手+分钟（api-football events）；
  2. 「明天世界杯有什么比赛」→ 「未开赛1场：西班牙 vs 阿根廷（03:00 开球）」；
  3. 「这场比赛你预测谁会赢」→ 锚定真实决赛（西vs阿）接地预测，首句自带
     「尚未开赛（7月20日凌晨3:00开球）」时效框定，法英幻觉对阵消失；
  4. 时效变体「今天这场比赛你预测谁会赢」→ 「这场比赛已经踢完了，结果是：…4-6…」
     结构化赛果卡，零检索。

## 4. 边界与刻意不做

- 「这场」多场同日的消歧（按焦点队名/最近提及场次）——保守回列表，等真实 badcase。
- 完赛短路只在**唯一一场**时触发；多场全完赛的预测句仍走检索（指代对象不明，不猜）。
- 非足球赛事（网球/电竞）的「这场谁会赢」：guard 让位 LLM 自由路由，联赛表外域
  解析不出锚点时按原话检索——结构化域仅覆盖 api-football 联赛表。
