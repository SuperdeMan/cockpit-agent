## 1. 背景

`agents/info/src/agent.py` 1269 行、~60 方法一个 `InfoAgent` 类，五域（天气/搜索/新闻/赛事/股票）+
早报 + 画像排序混在一起，域间无隔离——「改新闻碰坏赛事」的风险真实（审计 D6）。

## 2. 方案（mixin 拆域，零行为变化）

按域拆成 mixin 类，`InfoAgent` 多继承组合。**域方法照旧用 `self.` 相互调用，靠 MRO 透明解析——
逻辑逐字不动**，这是拆巨类最低风险的方式（相较把 self 状态改成参数传递）。

`agents/info/src/handlers/`：
- `_util.py` — 跨域共享工具（`_to_simplified` 繁简 / `_shanghai_now` / `_is_coordinate_label`），
  无 agent 依赖，避免循环导入。
- `weather.py` `WeatherMixin` — 天气/预报/预警/生活指数/空气质量。
- `search.py` `SearchMixin` — 通用检索 + 接地合成（+ `_plan_search`/`_is_fresh_sensitive` 及常量）。
- `sports.py` `SportsMixin` — 赛程/比分/进球详情/射手榜（+ `_detect_league`/`_ordinal_index`/
  `_sports_date`/`_season_candidates` 及联赛/提示常量）。
- `news.py` `NewsMixin` — 聚合/时效质量重排/逐条摘要/个性化/深挖桥接（+ 9 个新闻 helper 及常量）。
- `stock.py` `StockMixin` — 行情+K线（Tushare→东方财富降级）。
- `briefing.py` `BriefingMixin` — 主动早报（`on_start`→NATS→`agent.proactive`；经 self 调 NewsMixin）。

`agent.py`（1269 → **123 行**）：只留 `handle` 意图分发、公共件（`_resolve_city`/`_display_city`/
`_location_accuracy_note`）、provider 装配（`__init__`）；**文件尾向后兼容重导出**历史上定义于本文件、
被测试直接导入的模块级 helper（`_shanghai_now`/`_plan_search`/`_season_candidates`/`_is_fresh_sensitive`/
`_extract_news_subject`/`_news_interest_keywords`/`_rank_news_by_interest`/`_normalize_publish_time`/
`_rank_news_quality`/`_recent_only`），使 `from agents.info.src.agent import X` 路径不变、**测试零改动**。
